JsonPages Platform

Architecture Roadmap v3.0 | Target: Abstract Headless CMS
PHASE 1

Scaffolding & Data Structure

DevOps
Create the monorepo container and define the "Dual-Domain" data schema to support abstraction.
  • Execution of ngNest-init.sh to generate Angular + NestJS structure.
  • Implementation of the "File-Based DB" structure:
backend/data/ ├── content/ <-- Domain: Admin EDITOR (Items, Posts) │ ├── items.json │ └── posts.json └── config/ <-- Domain: Admin SETUP (Theme, Identity) ├── site.json (Title, Logo, Meta) ├── theme.json (Colors, Fonts, Layout) └── menu.json (Navigation Structure)
PHASE 2

Backend Core & Generics

NestJS
Build an agnostic engine that doesn't know the "Team" domain, only "Collections" and "Configurations".
  • Repository Pattern: IDataRepository interface with generic methods (read/write).
  • Service: JsonFsService (Node.js fs implementation).
  • Controllers:
    - ContentController (API for /content/*)
    - ConfigController (API for /config/*)
PHASE 3

Frontend "Themable"

Angular
Abstract UI that "draws itself" by reading the JSON configuration.
  • Config Service: APP_INITIALIZER loads theme.json and injects CSS variables.
  • Generic Components: Transform TeamComponent into CardComponent.
  • Dynamic Routing: Menu generated from menu.json.
PHASE 4

Dual Admin Suite

Angular
Clear separation between daily content management and platform setup.
  • Admin EDITOR: Dashboard to manage collections in content/.
  • Admin SETUP: Panel to modify theme.json (colors) and site.json (identity).
PHASE 5

Interactive Console

Bash/CLI
Unify operations, documentation, and onboarding in the terminal.
  • Tool: jp-console.sh
  • [TOUR]: Explains architecture (Content vs Config).
  • [RUN]: Automated startup.
  • [SCAFFOLD]: Empty JSON files generator.
